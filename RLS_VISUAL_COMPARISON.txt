The remote database appears to have been manually modified, removing all
service_role policies. This suggests someone tried to "improve security" by
restricting backend access, but instead BROKE security by:

  1. Making Edge Functions non-functional (403 Forbidden everywhere)
  2. Creating a privilege escalation vulnerability (users can create artifacts)
  3. Disabling rate limiting (anyone can spam the API)
  4. Breaking audit logging (cannot track usage)

service_role policies are NOT a security weakness - they are ESSENTIAL for
backend operations. The service_role key is kept SECRET and only used by
Edge Functions running on Supabase infrastructure.

Think of it like sudo in Linux:
  - Regular users have limited permissions (good)
  - System processes need elevated access (necessary)
  - service_role = sudo for Edge Functions

Without service_role policies, Edge Functions are like trying to run system
services without sudo - everything fails with "Permission Denied".
